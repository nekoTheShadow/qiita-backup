    「点字メーカープログラム」をSQLで回答したい

    はじめに

    ある日、Qiitaをながめていたところ、こんな面白そうなアドベントカレンダーを発見。

    https://qiita.com/advent-calendar/2021/ruby-challenge

    ぜひ参加してみたかったのですが、時すでに遅しで、参加枠はゼロ。そういうわけでアドベントカレンダー終了後に勝手に参加しようと思います。勝手に参加するのであれば、言語も好き勝手に選んでいいよねということで、今回はSQLを選択しました。なお稼働確認したDBMSはMySQL8.0.23です。

    回答

    回答となるクエリは以下の通りです。もうちょっとシンプルに書けそうではありますね。

    WITH RECURSIVE
BOIN (rome, bit) AS (
            SELECT 'A', 0b100000+0
  UNION ALL SELECT 'I', 0b101000+0
  UNION ALL SELECT 'U', 0b110000+0
  UNION ALL SELECT 'E', 0b111000+0
  UNION ALL SELECT 'O', 0b011000+0
),
SHIN (rome, bit) AS (
            SELECT 'K', 0b000001+0
  UNION ALL SELECT 'S', 0b000101+0
  UNION ALL SELECT 'T', 0b000110+0
  UNION ALL SELECT 'N', 0b000010+0
  UNION ALL SELECT 'H', 0b000011+0
  UNION ALL SELECT 'M', 0b000111+0
  UNION ALL SELECT 'R', 0b000100+0
),
REIGAI (rome, bit) AS (
            SELECT 'YA', 0b010010+0
  UNION ALL SELECT 'YU', 0b010011+0
  UNION ALL SELECT 'YO', 0b010110+0
  UNION ALL SELECT 'WA', 0b000010+0
  UNION ALL SELECT 'N' , 0b000111+0
),
HENKAN (rome, bit)  AS (
            SELECT rome, bit FROM BOIN
  UNION ALL SELECT rome, bit FROM REIGAI
  UNION ALL SELECT CONCAT(S.rome, B.rome), S.bit | B.bit FROM SHIN S CROSS JOIN BOIN B
),
INPUT (txt) AS (
            SELECT 'A HI RU'
  UNION ALL SELECT 'KI RI N'
  UNION ALL SELECT 'SI MA U MA'
  UNION ALL SELECT 'NI WA TO RI'
  UNION ALL SELECT 'HI YO KO'
  UNION ALL SELECT 'KI TU NE'
),
WORD (txt, lvl, rome, rest) AS (
    SELECT txt, 0, '  ', CONCAT(txt, ' ') FROM INPUT
  UNION ALL 
    SELECT txt, lvl + 1, SUBSTR(rest FROM 1 FOR INSTR(rest, ' ')), SUBSTR(rest FROM INSTR(rest, ' ') + 1)
    FROM WORD
    WHERE rest <> ''
),
A (txt, lvl, tenji) AS (
  SELECT W.txt, W.lvl, REPLACE(REPLACE(LPAD(BIN(H.bit), 6, '0'), '1', 'o'), '0', '-')
  FROM WORD W
  JOIN HENKAN H ON W.rome = H.rome
),
B (txt, lvl, col, tenji) AS (
            SELECT txt, lvl, 1, SUBSTR(tenji FROM 1 FOR 2) FROM A
  UNION ALL SELECT txt, lvl, 2, SUBSTR(tenji FROM 3 FOR 2) FROM A
  UNION ALL SELECT txt, lvl, 3, SUBSTR(tenji FROM 5 FOR 2) FROM A
),
C (txt, col, tenji) AS (
  SELECT txt, col, GROUP_CONCAT(tenji ORDER BY lvl SEPARATOR ' ')
  FROM B
  GROUP BY txt, col
)
SELECT txt, GROUP_CONCAT(tenji ORDER BY col SEPARATOR '\n') AS tenji
FROM C
GROUP BY txt

    簡単な解説

    やや長大なクエリですが、おおよそ3つの部分に分けられます。

    まず今回の変換対象である点字ですが、3種類に分けられます: ①母音だけからなるもの、②母音と子音を組み合わせたもの、③それ以外のもの。①はBOIN表、③はREIGAI表にまとめており、②については子音のみをSHIN表にまとめています。BOIN表とSHIN表とREIGAI表を組み合わせて、すべてのローマ字と点字を1対1で格納するHENKAN表を作成します。なおHENKAN表では点字を2進数(bit)で持っています。これは②で母音と子音を組み合わせる際に都合がよいからです (bit演算を利用している)。

    INPUT表には点字に変換したいローマ字列がまとめられています。与えられる文字列は空白区切りになっているので、トークンごとに分割してやる必要があります。この分割結果が格納されているのがWORD表で、再帰SQLを活用しています。

    あとはWORD表とHENKAN表を組み合わせて、文字列を求める形式に整形していきます。これをA表、B表、C表、そして最後のSELECT文で行っているわけですね。
