    String#slice!(nth)はIntegerではなく、Stringを返す。

    注意: この記事は2016/09/19現在のものです。またRubyのversionは2.3.0です。

    String#slice!は文字列を破壊的に変更するメソッドで、Rubyの日本語マニュアルである 「るりま」には次のように記述されています。

    slice!(nth) -> Integer
slice!(pos, len) -> String
slice!(substr) -> String
slice!(regexp, nth = 0) -> String
slice!(first..last) -> String
slice!(first...last) -> String
slice!(regexp, nth = 0) -> String

    要するにString#sliceの引数にひとつの整数を与えたときは、メソッドの戻り値は整数になると「るりま」には記述されているわけですが……

    irb(main):001:0> string = "ABCDEF"
=> "ABCDEF"
irb(main):002:0> string.slice!(0)
=> "A"
irb(main):003:0> string
=> "BCDEF"

    文字列が戻り値になっているような気がします。確かにオーバーロード(?)されたメソッドで、戻り値が違うというのは少し違和感がありますね。

    ちなみに英語のRubyドキュメントをみたところString#slice!(nth)でも文字列が返されるというような記述になっています。

    slice!(fixnum) → new_str or nil 
slice!(fixnum, fixnum) → new_str or nil 
slice!(range) → new_str or nil 
slice!(regexp) → new_str or nil 
slice!(other_str) → new_str or nil
