    相関サブクエリや共通表式を使って、他のテーブルをもとにUPDATEする

    前提

    商品在庫を管理するシステム構築にあたり、まず在庫数を商品ごとに格納しているinventoryテーブルがあるとします。

    goodspieces
    夏タイヤ10
    冬タイヤ20
    ホイール30

    次に商品がいつどれだけ入荷されるかの履歴や予定を格納しているarrivalテーブルがあります。

    goodscreated_atpieces
    夏タイヤ2021/03/311
    冬タイヤ2021/03/312
    ホイール2021/03/313
    夏タイヤ2021/04/014
    夏タイヤ2021/04/015
    冬タイヤ2021/04/016

    ここで2021/04/01になったので、2021/04/01に到着する商品数を在庫数に足してやる必要があります。つまり、arrivalテーブルから2021/04/01に到着する商品を取り出して、inventoryテーブルのpicesに足しこむようなUPDATE文を作る必要があります。

    なお実行後のinventoryテーブルは以下のようになるはずです。

    goodspieces
    夏タイヤ19
    冬タイヤ26
    ホイール30

    解法1

    もっともべたに書くと次のようなUPDATE文になるはずです。

    UPDATE inventory
SET pieces = inventory.pieces + (
  SELECT SUM(pieces) 
  FROM arrival
  WHERE inventory.goods = arrival.goods
  AND arrival.created_at = '2021-04-01')
WHERE EXISTS (
  SELECT *
  FROM arrival
  WHERE inventory.goods = arrival.goods
  AND arrival.created_at = '2021-04-01'
);

    解法2

    共通表式を利用すると、もう少しすっきりした見た目にはなります。

    WITH t AS (
  SELECT goods, SUM(pieces) AS total
   FROM arrival
  WHERE created_at = '2021-04-01'
  GROUP BY goods
)
UPDATE inventory
SET pieces = pieces + (SELECT total FROM t WHERE t.goods = inventory.goods)
WHERE EXISTS (SELECT * FROM t WHERE t.goods = inventory.goods);

    解法3

    解法3はこの問題特有の解法になります。到着する荷物がない場合「UPDATE対象にしない」と考えるのではなく、「0を足す」という風に考えると、WHERE節を省くことができます。解法1と解法2はWHERE節とSET節両方でサブクエリを書く必要があって、このせいで冗長な書きぶりになっていましたが、以下のクエリであればそれを避けることができます。

    UPDATE inventory
SET pieces = inventory.pieces + (
  SELECT COALESCE(SUM(pieces), 0)
  FROM arrival
  WHERE inventory.goods = arrival.goods
  AND arrival.created_at = '2021-04-01'
);
