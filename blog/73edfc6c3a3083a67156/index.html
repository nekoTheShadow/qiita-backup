    reduce/injectで処理を中断するにはbreakを使う(+三項演算子の話)

    注意: Rubyのバージョンは2.3.0です。それ以前のバージョンあるいはそれ以後のバージョンでは挙動が変化している可能性があります。

    reduceまたはinjectによる畳み込み演算の途中で処理を抜けたい場合はbreakを利用します。このときreduce/injectの戻り値にしたい値をbreakの引数に渡すことが重要です。

    irb(main):001:0> langs = %w(ruby perl python php java)
=> ["ruby", "perl", "python", "php", "java"]
irb(main):002:0> langs.reduce do |str, lang|
irb(main):003:1*   break str if lang.size < 4
irb(main):004:1>   "#{lang} / #{str}"
irb(main):005:1> end
=> "python / perl / ruby"

    なお引数なしのbreakを宣言した場合、戻り値はnilになります。

    irb(main):001:0> langs =%w(ruby perl python php java)
=> ["ruby", "perl", "python", "php", "java"]
irb(main):002:0> langs.reduce do |str, lang|
irb(main):003:1*   break if lang.size < 4
irb(main):004:1>   "#{lang} / #{str}"
irb(main):005:1> end
=> nil

    さて上のようなケースでは三項演算子を利用したいところです。三項演算子の真式もしくは偽式にbreakを利用する場合はbreakを(break str)のように書く必要があります。

    irb(main):008:0> langs =%w(ruby perl python php java)
=> ["ruby", "perl", "python", "php", "java"]
irb(main):009:0> langs.reduce{|str, lang| lang.size < 4 ? (break str) : "#{lang} / #{str}"}
=> "python / perl / ruby"

    なおbreakを()でくくらないと例外が発生します。

    irb(main):013:0> langs = %w(ruby perl python php java)
=> ["ruby", "perl", "python", "php", "java"]
irb(main):014:0> langs.reduce{|str, lang| lang.size < 4 ? break str : "#{lang} / #{str}"}
SyntaxError: (irb):14: syntax error, unexpected tIDENTIFIER, expecting ':'
...lang| lang.size < 4 ? break str : "#{lang} / #{str}"}
...                               ^
        from C:/Ruby23/bin/irb.cmd:19:in `<main>'

    ではbreakに引数を渡さない場合はどうでしょうか? 実は例外は発生しません。

    irb(main):015:0> langs = %w(ruby perl python php java)
=> ["ruby", "perl", "python", "php", "java"]
irb(main):016:0> langs.reduce{|str, lang| lang.size < 4 ? break : "#{lang} / #{str}"}
=> nil

    このあたりのさじ加減(?)はなかなか難しいところがあります。まあbreakやreduce/injectを多用するプログラミングスタイルはわかりづらいので避けるべきという話かもしれませんが(´・ω・`)
