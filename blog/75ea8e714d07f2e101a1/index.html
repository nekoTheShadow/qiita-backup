    javacの警告が100個までしか出力されない。

    この記事は以下の環境で動作確認を行いました。ただし、ほかのバージョンやほかのベンダーでも通用する内容だとは思います(´・ω・｀)

    C:\jdk>java -version
openjdk version "12" 2019-03-19
OpenJDK Runtime Environment (build 12+33)
OpenJDK 64-Bit Server VM (build 12+33, mixed mode, sharing)

C:\jdk>javac -version
javac 12

    たとえば以下のようなコードがあったとします。

    import java.util.List;

public class Sample {
    public static void main(String[] args) {
        List list001;
        List list002;
        List list003;
        List list004;
        List list005;

        // 長いので中略

        List list196;
        List list197;
        List list198;
        List list199;
        List list200;
    }
}

    煩雑なので一部省略していますが、Listが原型(Raw Type)のまま200個宣言されています。よってこれ-Xlintオプション付きでコンパイルしたとき、200個の警告が出力されてほしいのですが、実際は100個しか出力されません。

    コンパイル時警告の最大数は-Xmaxwarnsオプションにより設定することができます。この値がデフォルトでは100になっているらしく、そのために上記の例では100個しか警告が出力されませんでした。よって「警告を200個出力したい」、もとい「警告を無制限に出力したい」場合は-Xmaxwarnsオプションに巨大な値を設定すればよいということです。

    
C:\jdk>javac -Xlint -Xmaxwarns 999999 Sample.java
